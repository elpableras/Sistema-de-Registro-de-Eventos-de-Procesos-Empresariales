Registro General

Está pantalla consta de dos pestañas, una de ellas muestra la información "General" del Proceso, dividida en varias partes:

Proceso General
Nombre Descripción corta del Proceso.
Código General

Cada proceso tiene un código de identificación general, este código es idéntico para todos los procesos de este mismo tipo.
El código del proceso identifica la familia de procesos del mismo tipo y que se generan ante un mismo evento.

El código estará formado por:
Cód. Procedimiento + código correlativo de proceso dentro de procedimiento

Descripción Descripción detallada, indicando el momento y el evento o eventos que provocan la creación del proceso.
Tipo

Dos tipos:

  • E- Proceso electrónico
  • P- Proceso en papel

Código del Procedimiento

Código del procedimiento al que pertenece el proceso o al procedimiento en que se genera.

Actividad Actividad del proceso donde se realiza el proceso (Ejemplo: Actividad 6 del Procedimiento “Comprobación del Plan de Acogida”).
Documentación Vinculada

Lista de documentos (físicos o digitales) relacionados con el proceso .

  1. Código de documento o enlace al mismo si es digital
  2. Tipo y descripción del tipo de documento

Si la lista de documentos es fija, la descripción de los mismos [b)] iría a este nivel,
si depende de los datos que se generan en el evento en que se produce el proceso, irán en el atributo [3] del proceso especifico.

Nota:
Recomendamos todos los procesos digitales. Y si el proceso es papel, se genera un proceso digital que referencia al de documento.

Lista campos

Lista y tipo de los campos que van a tomarse en el proceso especifico.

  • Nombre (TemperaturaAmbiente)
  • Descripción (Temperatura ambiente del vehículo)
  • Tipo (número con dos decimales)
  • Unidad (grados Celsius)
Puede ser que sólo se guarde la referencia a la documentación asociada, pero normalmente sería interesante guardar información resumen
de la documentación (siempre que sea factible generar esta información automáticamente).
Frecuencia Frecuencia de la toma del proceso (Ejemplo: cada hora, cada vez que ocurra un evento, …).
Método Método de captura del proceso y/o de la información adicional al proceso (Ejemplo: por formulario, por sonda de temperatura,
pistola de código de barras, mezcla de varios métodos, … ).
Lista Indicadores

Indicadores a aplicar al proceso (Ejemplo: tiempo entre procesos, media de temperatura registrada,…).

Generador del Proceso

Persona o empresa externa o sistema que genera el proceso.

Esta información podría estar a nivel de proceso específico si puede variar el agente que genera el proceso [4].
Lo habitual es que sea siempre el generador de proceso sea siempre mismo y este a este nivel.

Autorizados para su consulta

Persona, empresa externa o sistema autorizados para la consulta del proceso.

Esta información podría estar a nivel de proceso específico si puede variar el/los agente/s que consulta/n el proceso [5].
Lo habitual es que sea siempre el mismo y este a este nivel.

Y la otra pestaña, se accede a ella pulsando en el botón "Siguiente" y muestra el Ciclo de Vida del proceso

Proceso Ciclo de Vida
Archivo Se considera “Archivo” a la ubicación donde se guardan los procesos activos.

Todos los procesos de calidad deberán ser archivados física o electrónica mediante un método que mejore la accesibilidad
y la recuperación para un usuario. Si se utilizan archivos electrónicos, se deben implementar algún sistema de copia de seguridad
u otras medidas adecuadas para evitar la pérdida de los procesos.

Los procedimientos deberán definir la ubicación donde los procesos de la Calidad se guarden.
Retención El tiempo de “retención” de un proceso de calidad se refiere a cuánto tiempo debe ser mantenido hasta que es descartado o destruido,
o enviado a un almacén de larga duración. Los procesos podrían ser retenidos por más tiempo del mínimo por conveniencia de la
organización.

Los tiempos de retención para los procesos de calidad específicos de la organización deben ser determinados por la propia organización y
recogidos en los documentos del Sistema de Calidad.

Puede haber reglas generales de retención o especificas (Ejemplo: proceso almacenado hasta que se jubile un conductor + un periodo
adicional de 5 años).
Conservación Todos los procesos de calidad deberán ser archivados y almacenados en el entorno de la oficina a menos que se especifiquen medios
concretos y/o controles especiales del entorno para prevenir daños, deterioro o pérdida, siempre que se trate de documentación física.

Y si se almacena de manera digitalizada, se establecerán los mecanismos para la conservación y consulta del mismo.
Disposición Procedimientos definirán como los procesos de calidad serán dispuestos cuando se exceda el tiempo de retención.

La disposición podrá ser descartar o destruir los procesos de calidad, o contactar con la Oficina de Gestión de procesos para recibir
instrucciones para el almacenamiento de larga duración.
Almacén Después del periodo de retención in-situ, todos los procesos de calidad deberán ser almacenados de forma indefinida según lo defina
el Responsable de Gestión de procesos.

Existe al final un CheckBox con la etiqueta "Versión Modificada", que se activara automáticamente cuando el proceso sea traído del Servidor,
para ser cambiado.

Finalmente para mandar el proceso con los datos a almacenar a la BBDD del Servidor, se oprimirá el botón "Enviar",
también existe otro botón para ir hacia la parte de la Información General, el boton "Atras".
